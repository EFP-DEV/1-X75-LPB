## Version 1 — “Directe / formules en dur”

```text
Variable n en entier
Variable montant en decimal

Début
    Ecrire "Nombre de photocopies ?"
    Lire n

    Si n <= 10 Alors
        montant <- n * 0,20
    SinonSi n <= 30 Alors
        montant <- 10 * 0,20 + (n - 10) * 0,10
    Sinon
        montant <- 10 * 0,20 + 20 * 0,10 + (n - 30) * 0,05
    FinSi

    Ecrire "Montant total : ", montant, " €"
Fin
```

### Idée

On applique directement les **3 cas** du problème (≤10, ≤30, >30) avec les formules correspondantes.

### Points forts

* Très lisible pour débuter : on voit immédiatement les trois tranches.
* Peu de variables : simple.

### Limites

* **Beaucoup de “valeurs en dur”** (10, 20, 30, 0,20, 0,10, 0,05).
* Si la tarification change, il faut modifier plusieurs endroits → risque d’erreur.
* On “répète” la logique de base (10 premières + 20 suivantes…) dans chaque formule.

## Version 2 — Cumul “en dur”

**Idée :** on calcule d’abord ce qui dépasse 30, puis ce qui dépasse 10, puis le reste.
**Atout :** pas de grosse formule unique, on cumule par étapes.
**Limite :** seuils et prix sont en dur, et on écrase `n`.

```text
Variable n en entier
Variable montant en decimal

Début
    Ecrire "Nombre de photocopies ?"
    Lire n

    montant <- 0

    Si n > 30 Alors
        montant <- montant + (n - 30) * 0,05
        n <- 30
    FinSi

    Si n > 10 Alors
        montant <- montant + (n - 10) * 0,10
        n <- 10
    FinSi

    montant <- montant + n * 0,20

    Ecrire "Montant total : ", montant, " €"
Fin
```

## Version 3 — Cumul + tranches en variables

**Ce qui change par rapport à la v2 :**

* On garde exactement la **même stratégie de cumul**.
* On remplace **10 et 30** par `tranche1` et `tranche1 + tranche2`.
* Le code devient **paramétrable** : si les tranches changent, tu modifies seulement les variables.

**Ce qui ne change pas :**

* Les prix sont encore en dur.
* On écrase encore `n` (comme en v2).

```text
Variable n en entier
Variable montant en decimal
Variable tranche1, tranche2 en entier

Début
    Ecrire "Nombre de photocopies ?"
    Lire n

    tranche1 <- 10
    tranche2 <- 20
    montant <- 0

    Si n > tranche1 + tranche2 Alors
        montant <- montant + (n - (tranche1 + tranche2)) * 0,05
        n <- tranche1 + tranche2
    FinSi

    Si n > tranche1 Alors
        montant <- montant + (n - tranche1) * 0,10
        n <- tranche1
    FinSi

    montant <- montant + n * 0,20

    Ecrire "Montant total : ", montant, " €"
Fin
```

## Version 4 — Cumul + tranches + prix en variables

**Ce qui change par rapport à la v3 :**

* On garde le **cumul**.
* On met aussi les **prix** en variables (`prix1`, `prix2`, `prix3`).
* Résultat : toute la tarification est dans des variables → le calcul ne dépend plus de nombres “magiques”.

**Pourquoi c’est mieux :**

* Maintenance : changer un tarif = changer une variable.
* Lisibilité : `prix1`/`prix2`/`prix3` disent clairement ce qu’on fait.

```text
Variable n en entier
Variable montant en decimal
Variable tranche1, tranche2 en entier
Variable prix1, prix2, prix3 en decimal

Début
    Ecrire "Nombre de photocopies ?"
    Lire n

    tranche1 <- 10
    tranche2 <- 20

    prix1 <- 0,20
    prix2 <- 0,10
    prix3 <- 0,05

    montant <- 0

    Si n > tranche1 + tranche2 Alors
        montant <- montant + (n - (tranche1 + tranche2)) * prix3
        n <- tranche1 + tranche2
    FinSi

    Si n > tranche1 Alors
        montant <- montant + (n - tranche1) * prix2
        n <- tranche1
    FinSi

    montant <- montant + n * prix1

    Ecrire "Montant total : ", montant, " €"
Fin
```

---

### Remarque pédagogique (utile en cours)

Ces versions (2 → 3 → 4) illustrent une vraie progression :

1. **Technique** (cumul par étapes),
2. **Paramétrage des tranches** (plus flexible),
3. **Paramétrage complet (tranches + prix)** (plus maintenable).

Si tu veux une v5 “cumul propre” qui **n’écrase pas `n`** (on utilise `reste`), je te la fais aussi.
