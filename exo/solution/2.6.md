# Exercice 2.6

## Introduction

Dans cet exercice, tu dois écrire un algorithme qui :

1. **demande** une durée en minutes ;
2. **calcule** combien cela fait en heures et minutes ;
3. **affiche** le résultat sous la forme `Xh Ymin`.

Une variable est comme une **boîte** :

* quand on fait une **affectation** (`<-`), on met une valeur dans la boîte ;
* `Lire` met une valeur dans une variable ;
* `Ecrire` affiche, **sans modifier** les variables.

---

## Objectifs

Comprendre que :

* 60 minutes = 1 heure ;
* on peut obtenir le nombre d’heures avec `/ 60` (en entier) ;
* on peut obtenir les minutes restantes en retirant `heures * 60`.

---

## Pseudo-code

```
1   Variables minutes, heures, reste en Entier
2
3   Début
4       Ecrire "Durée en minutes : "
5       Lire minutes
6
7       heures <- minutes / 60
8       reste <- minutes - (heures * 60)
9
10      Ecrire minutes & " minutes = " & heures & "h " & reste & "min"
11  Fin
```

---

## Syntaxe

* **Division** : `/`
  → Donne le quotient. Ici, comme `heures` est un **Entier**, on garde seulement la partie entière.
  Exemple : `135 / 60 = 2` (en entier).

* **Calcul des minutes restantes** : `reste <- minutes - (heures * 60)`
  → On enlève les minutes déjà “rangées” en heures.
  Exemple : `reste = 135 - (2 * 60) = 15`.

* **Concaténation** : `&`
  → Permet de coller du texte et des valeurs dans une même phrase.

* **Afficher** :
  `Ecrire minutes & " minutes = " & heures & "h " & reste & "min"`
  → Affiche le résultat dans une phrase.

## Note pédagogique : arrondi / troncation

Quand on écrit :

```
heures <- minutes / 60
reste  <- minutes - (heures * 60)
```

cela suppose que `heures` vaut le **nombre d’heures complètes**.

Donc :

* si `heures` est un **Entier**, alors `minutes / 60` est **tronqué** (on garde la partie entière)
  → c’est bon.
* si `heures` est un **Numérique** (décimal), alors `minutes / 60` peut être **2,25**
  → et là `reste` devient **0** (car 135 - 2,25×60 = 135 - 135 = 0), ce qui est faux.


### Troncation vs arrondi

* **Troncation** (ce qu’on veut) : on coupe les décimales.
  Exemple : 135 / 60 = 2,25 → on garde **2**.

* **Arrondi** (dangereux ici) : on arrondit à l’entier le plus proche.
  Exemple : 89 / 60 = 1,48 → arrondi = **1** (ok)
  mais 91 / 60 = 1,51 → arrondi = **2** (pas ok !)
  et alors :

  ```
  reste = 91 - (2 * 60) = -29
  ```

  → minutes restantes négatives : impossible.

Donc on ne veut **jamais** un arrondi, on veut une **troncation** (ou “partie entière”).


### Conclusion pratique 

Pour que la méthode `/ 60` puis `- * 60` soit fiable :

1. `heures` doit être déclaré en **Entier** (comme tu l’as fait) ;
2. la division `/` doit produire un résultat **tronqué** quand on le met dans un Entier.

Si ton langage/pseudo-langage ne garantit pas ça, il faut écrire explicitement la troncation, par exemple :

* `heures <- ENTIER(minutes / 60)`
  ou
* `heures <- TRONC(minutes / 60)`
