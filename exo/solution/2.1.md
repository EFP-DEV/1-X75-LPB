# Exercice 2.1

## Introduction

Dans cet exercice, tu dois **suivre l’algorithme ligne par ligne** et repérer ce qui est **affiché** à l’écran.

Une variable est comme une **boîte** :

* quand on fait une **affectation** (`<-`), on met une valeur dans la boîte ;
* `Ecrire` affiche une valeur, **mais ne change pas** la variable.

---

## Objectifs

Comprendre que :

* une variable peut être utilisée dans un calcul ;
* `Ecrire` affiche la valeur actuelle d’une variable ;
* afficher une valeur ne modifie pas les variables.

---

## Pseudo-code

```
1   Variables val, double en Numérique
2
3   Début
4       val <- 231
5       double <- val * 2
6       Ecrire val
7       Ecrire double
8   Fin
```

---

## Syntaxe

* **Déclaration** : `Variables x, y en Numérique`
  → On annonce les variables et leur type (ici : nombres).

* **Affectation** : `x <- 231`
  → On met 231 dans la variable `x`.

* **Calcul** : `double <- val * 2`
  → On prend la valeur actuelle de `val`, on la multiplie par 2, puis on met le résultat dans `double`.

* **Affichage** : `Ecrire val`
  → On affiche la valeur actuelle de `val` (cela ne modifie rien).

* **Ordre des lignes** : on exécute **de haut en bas**.

## Idée essentielle

`Ecrire` ne calcule pas, ne copie pas, ne modifie pas :
il **affiche** simplement la valeur **au moment où la ligne est exécutée**.

---

## Image mentale

Les variables sont des boîtes.
`Ecrire` c’est comme **regarder l’étiquette** et lire le contenu à voix haute :
la boîte reste identique.

---

## Conclusion

Le programme affiche :

```
231
462
```
