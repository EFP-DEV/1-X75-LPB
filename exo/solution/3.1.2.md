# Exercice 3.1 — Version 2 (sans `Sinon`, avec deux tests)

## Introduction

Dans cette deuxième version, on garde l’idée de ne pas utiliser `Sinon`, mais on corrige le défaut de la version 1 : on veut aussi afficher un message quand le nombre est négatif.

On ne traite pas le zéro (on suppose que l’utilisateur n’entre pas 0).

---

## Objectifs

Comprendre que :

* on peut gérer deux cas sans `Sinon` en écrivant deux tests séparés ;
* chaque test a sa propre condition ;
* si les conditions sont bien choisies, un seul message s’affiche.

---

## Pseudo-code (Version 2)

```
1   Variables n en Numérique
2
3   Début
4       Ecrire "Entrez un nombre (non nul) : "
5       Lire n
6
7       Si n > 0 Alors
8           Ecrire "Le nombre est positif."
9       FinSi
10
11      Si n < 0 Alors
12          Ecrire "Le nombre est négatif."
13      FinSi
14  Fin
```

---

## Syntaxe

* `n > 0` : condition vraie si `n` est positif.
* `n < 0` : condition vraie si `n` est négatif.

Ici, on a deux `Si` séparés.
Chaque `Si` n’affiche un message que si sa condition est vraie.

---

## Lecture en français

| Ligne | Action décrite                                                |
| ----: | ------------------------------------------------------------- |
|     1 | Je déclare une variable `n`.                                  |
|     4 | Je demande un nombre non nul.                                 |
|     5 | Je lis ce nombre et je le stocke dans `n`.                    |
|     7 | Je teste la condition `n > 0`.                                |
|     8 | Si la condition est vraie, j’affiche “Le nombre est positif.” |
|    11 | Je teste la condition `n < 0`.                                |
|    12 | Si la condition est vraie, j’affiche “Le nombre est négatif.” |
|    14 | Je termine l’algorithme.                                      |

---

## Inspection des valeurs

### Exemple 1 — si l’utilisateur tape 7

| Test | Condition | Résultat | Affichage              |
| ---- | --------- | -------- | ---------------------- |
| 1    | `n > 0`   | vrai     | Le nombre est positif. |
| 2    | `n < 0`   | faux     | (rien)                 |

Un seul message s’affiche : positif.

---

### Exemple 2 — si l’utilisateur tape -3

| Test | Condition | Résultat | Affichage              |
| ---- | --------- | -------- | ---------------------- |
| 1    | `n > 0`   | faux     | (rien)                 |
| 2    | `n < 0`   | vrai     | Le nombre est négatif. |

Un seul message s’affiche : négatif.

---

## Amélioration par rapport à la version 1

Version 1 : si le nombre est négatif, l’algorithme n’affichait rien.
Version 2 : on ajoute un second test, ce qui permet d’afficher un message dans les deux cas.

---

## Défaut restant

Cette version fonctionne parce qu’on suppose que l’utilisateur ne saisit pas 0.

Si l’utilisateur tape 0 :

* `n > 0` est faux
* `n < 0` est faux
  Donc aucun message ne s’affiche.

Ce défaut sera traité dans une version qui gère explicitement le zéro.

---

## Conclusion

La version 2 permet d’afficher :

* “positif” si `n > 0` est vraie ;
* “négatif” si `n < 0` est vraie ;
