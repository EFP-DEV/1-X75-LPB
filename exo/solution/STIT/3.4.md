# Exercice 3.4 — Version 1 (naïve : comme 3.1, sans gérer 0)

## Introduction

On reprend l’exercice 3.1, mais cette fois on doit aussi traiter le cas où le nombre vaut zéro.

Pour construire des versions et observer les défauts, on commence par une version identique à 3.1, qui ne gère pas correctement 0.

---

## Objectifs

Comprendre que :

* si on teste seulement `n > 0`, le `Sinon` regroupe tous les autres cas ;
* “tous les autres cas” inclut à la fois les nombres négatifs et le zéro ;
* pour traiter 0 correctement, il faut un test supplémentaire.

---

## Pseudo-code

```
1   Variables n en Numérique
2
3   Début
4       Ecrire "Entrez un nombre : "
5       Lire n
6
7       Si n > 0 Alors
8           Ecrire "Le nombre est positif."
9       Sinon
10          Ecrire "Le nombre est négatif."
11      FinSi
12  Fin
```

---

## Inspection des valeurs

### Exemple — si l’utilisateur tape 0

* `n > 0` est fausse
* on va dans `Sinon`
* affichage : “Le nombre est négatif.”

---

## Défaut de cette version

Cette version se trompe quand `n = 0`, car 0 n’est ni positif ni négatif.

---

## Conclusion

Cette version sert de point de départ : elle montre le problème.

Une version suivante devra distinguer trois cas :

* `n > 0` : positif
* `n < 0` : négatif
* sinon : zéro

---

# Exercice 3.4 — Version 2 (traitement correct de 0)

## Introduction

Dans cette version, on ajoute un test qui distingue explicitement le zéro.

---

## Pseudo-code

```
1   Variables n en Numérique
2
3   Début
4       Ecrire "Entrez un nombre : "
5       Lire n
6
7       Si n > 0 Alors
8           Ecrire "Le nombre est positif."
9       SinonSi n < 0 Alors
10          Ecrire "Le nombre est négatif."
11      Sinon
12          Ecrire "Le nombre est nul."
13      FinSi
14  Fin
```

---

## Lecture en français

| Ligne | Action décrite                                  |
| ----: | ----------------------------------------------- |
|     4 | Je demande un nombre.                           |
|     5 | Je lis le nombre et je le stocke dans `n`.      |
|     7 | Je teste si `n` est supérieur à 0.              |
|     8 | Si la condition est vraie, j’affiche “positif”. |
|     9 | Sinon, je teste si `n` est inférieur à 0.       |
|    10 | Si la condition est vraie, j’affiche “négatif”. |
|    11 | Sinon, cela signifie que `n` vaut 0.            |
|    12 | J’affiche “nul”.                                |

---

## Inspection des valeurs

### Exemple 1 — n = 7

* `n > 0` vrai → positif

### Exemple 2 — n = -3

* `n > 0` faux
* `n < 0` vrai → négatif

### Exemple 3 — n = 0

* `n > 0` faux
* `n < 0` faux
* donc on va dans le dernier `Sinon` → nul

---

## Amélioration par rapport à la version 1

Version 1 : rangeait le zéro dans “négatif”.
Version 2 : distingue correctement les trois cas.

---

## Conclusion

L’ajout de `SinonSi n < 0` permet de traiter le zéro correctement.

La structure finale est :

* si `n > 0` → positif
* sinon si `n < 0` → négatif
* sinon → nul
