# Exercice 4.5 — Version 5 (composition OU pour "battu")

## Introduction

Dans la version 4, "battu" apparaît deux fois :

* ligne 11 : un autre candidat a > 50 %
* ligne 13 : s1 < 12,5 %

Le message est le même. La raison est différente, mais le résultat est identique : le candidat 1 a perdu.

On peut regrouper ces deux cas en une seule condition avec `OU`.

---

## Pseudo-code (Version 5)

```
1   Variables s1, s2, s3, s4 en Décimal
2
3   Début
4       Lire s1
5       Lire s2
6       Lire s3
7       Lire s4
8
9       Si s1 > 50 Alors
10          Ecrire "élu"
11      SinonSi s1 < 12,5 OU s2 > 50 OU s3 > 50 OU s4 > 50 Alors
12          Ecrire "battu"
13      SinonSi s1 >= s2 ET s1 >= s3 ET s1 >= s4 Alors
14          Ecrire "ballottage favorable"
15      Sinon
16          Ecrire "ballottage défavorable"
17      FinSi
18  Fin
```

---

## Syntaxe

* `OU` : il suffit qu'**une seule** des conditions soit vraie pour entrer dans le bloc.
* L'ordre des termes dans le `OU` n'a pas d'importance logique (le résultat est le même).

---

## Lecture en français

| Ligne | Action décrite |
| ----: | -------------- |
|     9 | Le candidat 1 a-t-il plus de 50 % ? |
|    10 | Si oui : élu. |
|    11 | Sinon : a-t-il moins de 12,5 %, ou un autre candidat a-t-il plus de 50 % ? |
|    12 | Si oui : battu. |
|    13 | Sinon : est-il en tête (ou à égalité) ? |
|    14 | Si oui : ballottage favorable. |
|    16 | Sinon : ballottage défavorable. |

---

## Comparaison v4 / v5

| Version 4 (deux blocs) | Version 5 (OU) |
|-------------------------|----------------|
| "battu" apparaît deux fois | "battu" apparaît une fois |
| On voit clairement les deux raisons séparées | Plus compact, moins de lignes |
| Plus facile à déboguer (on sait quelle branche) | Plus difficile de savoir **pourquoi** battu |

Les deux sont correctes. En pratique, on choisit selon le besoin :

* si on veut **comprendre** ou **afficher la raison** → v4
* si on veut **un code court** et que la raison importe peu → v5

---

## Inspection des valeurs

### Exemple 1 — s1 = 30, s2 = 55, s3 = 10, s4 = 5

* `s1 > 50` : faux
* `s1 < 12,5` : faux, `s2 > 50` : vrai → **battu** ✓

### Exemple 2 — s1 = 8, s2 = 40, s3 = 30, s4 = 22

* `s1 > 50` : faux
* `s1 < 12,5` : vrai → **battu** ✓

### Exemple 3 — s1 = 35, s2 = 30, s3 = 20, s4 = 15

* `s1 > 50` : faux
* `s1 < 12,5` : faux, `s2 > 50` : faux, `s3 > 50` : faux, `s4 > 50` : faux
* `s1 >= s2 ET s1 >= s3 ET s1 >= s4` : vrai → **ballottage favorable** ✓

---

## Conclusion

La version 5 est la version finale. Quatre issues, quatre branches, pas de doublon :

1. `s1 > 50` → **élu**
2. `s1 < 12,5 OU s2 > 50 OU s3 > 50 OU s4 > 50` → **battu**
3. `s1 >= s2 ET s1 >= s3 ET s1 >= s4` → **ballottage favorable**
4. sinon → **ballottage défavorable**
