# Exercice 3.5 — Version 1 (comme 3.2 v2 : ne traite pas le produit nul)

## Introduction

On reprend l’exercice 3.2, mais cette fois on doit aussi traiter le cas où le produit peut être nul.

On ne doit toujours pas calculer `a * b`.

Pour construire des versions, on repart d’une version qui respecte la contrainte mais qui ne gère pas le produit nul.

---

## Objectifs

Comprendre que :

* le produit est nul dès qu’au moins un des deux nombres vaut 0 ;
* si on utilise seulement “même signe / signes différents” avec un `Sinon`, le cas 0 peut être mal classé ;
* il faut détecter le zéro avant de décider “positif ou négatif”.

---

## Pseudo-code

```
1   Variables a, b en Numérique
2
3   Début
4       Ecrire "Entrez un premier nombre : "
5       Lire a
6       Ecrire "Entrez un second nombre : "
7       Lire b
8
9       Si (a > 0 ET b > 0) OU (a < 0 ET b < 0) Alors
10          Ecrire "Le produit est positif."
11      Sinon
12          Ecrire "Le produit est négatif."
13      FinSi
14  Fin
```

---

## Inspection des valeurs

### Exemple — a = 0, b = 5

* `(a > 0 ET b > 0)` est faux
* `(a < 0 ET b < 0)` est faux
* on tombe dans `Sinon`
* affichage : “Le produit est négatif.”

---

## Défaut de cette version

Quand `a = 0` ou `b = 0`, le produit est nul.

Cette version affiche “négatif” alors que le produit est nul.

---

## Conclusion

Cette version sert de point de départ : elle respecte la contrainte (pas de multiplication) mais elle ne traite pas le cas du produit nul.

Une version suivante devra tester le zéro avant tout.

---

# Intermédiaire d’analyse (entre la version 1 et la version 2)

### 1) Nouvelle information à intégrer

Un produit est nul si au moins un facteur vaut 0 :

* si `a = 0` ou `b = 0` alors `a*b = 0`.

On n’a pas besoin de calculer le produit pour le savoir.

### 2) Ordre logique des tests

Pour être sûr de ne pas se tromper :

1. tester d’abord si `a = 0` ou `b = 0`
   dans ce cas, le produit est nul ;
2. sinon, décider entre positif et négatif avec les signes.

---

# Exercice 3.5 — Version 2 (traitement correct du produit nul, sans calcul)

## Introduction

Dans cette version, on ajoute un premier test : si un des deux nombres vaut 0, le produit est nul.

Sinon, on applique la règle des signes comme dans l’exercice 3.2.

---

## Pseudo-code

```
1   Variables a, b en Numérique
2
3   Début
4       Ecrire "Entrez un premier nombre : "
5       Lire a
6       Ecrire "Entrez un second nombre : "
7       Lire b
8
9       Si (a = 0) OU (b = 0) Alors
10          Ecrire "Le produit est nul."
11      SinonSi (a > 0 ET b > 0) OU (a < 0 ET b < 0) Alors
12          Ecrire "Le produit est positif."
13      Sinon
14          Ecrire "Le produit est négatif."
15      FinSi
16  Fin
```

---

## Syntaxe

* `OU` : au moins une des conditions est vraie.
* `(a = 0) OU (b = 0)` : au moins un des deux vaut zéro.
* `SinonSi` : test effectué seulement si le premier `Si` est faux.

---

## Inspection des valeurs

### Exemple 1 — a = 0, b = 5

* `(a = 0) OU (b = 0)` est vrai
  Affichage : Le produit est nul.

### Exemple 2 — a = -2, b = -3

* premier test (zéro) : faux
* même signe : vrai
  Affichage : Le produit est positif.

### Exemple 3 — a = -2, b = 3

* premier test (zéro) : faux
* même signe : faux
  Affichage : Le produit est négatif.

---

## Amélioration par rapport à la version 1

Version 1 : affichait “négatif” quand le produit était nul.
Version 2 : détecte d’abord le zéro, puis seulement ensuite décide positif/négatif.

---

## Conclusion

L’algorithme traite maintenant correctement les trois cas, sans calculer `a*b` :

* si `a = 0` ou `b = 0` → produit nul
* sinon si même signe → produit positif
* sinon → produit négatif
