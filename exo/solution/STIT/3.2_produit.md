# Exercice 3.2 — Version 1 (naïve, interdite par la contrainte)

## Introduction

On doit demander deux nombres à l’utilisateur et indiquer si leur produit est positif ou négatif.

On ignore le cas où le produit est nul, donc on suppose que l’utilisateur ne donne pas une combinaison qui rend le produit égal à 0.

Contrainte importante : on ne doit pas calculer le produit.

Pour montrer la logique des versions, on commence par une première version “naturelle”, puis on constatera son défaut.

---

## Objectifs

Comprendre que :

* le produit de deux nombres peut être positif ou négatif ;
* la contrainte interdit de calculer `a * b` ;
* une première version peut être correcte mathématiquement mais non conforme.

---

## Pseudo-code (Version 1)

```
1   Variables a, b, p en Numérique
2
3   Début
4       Ecrire "Entrez un premier nombre : "
5       Lire a
6       Ecrire "Entrez un second nombre : "
7       Lire b
8
9       p <- a * b
10
11      Si p > 0 Alors
12          Ecrire "Le produit est positif."
13      Sinon
14          Ecrire "Le produit est négatif."
15      FinSi
16  Fin
```

---

## Lecture en français

| Ligne | Action décrite                                  |
| ----: | ----------------------------------------------- |
|     4 | Je demande le premier nombre.                   |
|     5 | Je lis `a`.                                     |
|     6 | Je demande le second nombre.                    |
|     7 | Je lis `b`.                                     |
|     9 | Je calcule `p` comme le produit de `a` et `b`.  |
|    11 | Je teste si `p > 0`.                            |
|    12 | Si la condition est vraie, j’affiche “positif”. |
|    14 | Sinon, j’affiche “négatif”.                     |

---

## Inspection des valeurs

### Exemple 1 — a = 2, b = 3

* `p <- 2 * 3` donc `p = 6`
* `p > 0` est vraie
  Affichage : Le produit est positif.

### Exemple 2 — a = -2, b = 3

* `p <- -2 * 3` donc `p = -6`
* `p > 0` est fausse
  Affichage : Le produit est négatif.

---

## Défaut de cette version

Cette version viole la contrainte :

* on calcule explicitement `a * b` à la ligne 9, ce qui est interdit.

Même si le résultat est correct, la méthode n’est pas acceptable pour l’exercice.

---

## Conclusion

La version 1 sert à comprendre l’objectif, mais elle n’est pas conforme car elle calcule le produit.
Une version suivante devra déterminer le signe du produit uniquement en utilisant les signes de `a` et `b`, sans multiplication.
