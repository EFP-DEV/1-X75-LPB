# Exercice 4.5 — Version 3 (ballottage favorable / défavorable)

## Introduction

Le candidat 1 est en ballottage (il a ≥ 12,5 % mais pas > 50 %). Il faut maintenant distinguer :

* **favorable** : il est arrivé en tête (y compris en cas d'égalité)
* **défavorable** : il n'est pas en tête

---

## La question qu'il faut se poser

"Arrivé en tête" signifie quoi avec 4 candidats ?

L'énoncé précise : **y compris en cas d'égalité**. Donc "en tête" = son score est **supérieur ou égal** à celui de chaque autre candidat.

Ce qui donne : `s1 >= s2 ET s1 >= s3 ET s1 >= s4`.

Si un autre candidat a le même score, le candidat 1 est quand même considéré en tête. C'est le choix de l'énoncé.

---

## Pseudo-code (Version 3)

```
1   Variables s1, s2, s3, s4 en Décimal
2

4       Lire s1

14          Ecrire "ballottage favorable"

17      FinSi

```

---

## Remarque sur l'ordre des tests

On a inversé l'ordre de "battu" et "ballottage" par rapport à la version 2.

Pourquoi ? Parce qu'au moment de tester "en tête" (ligne 13), on veut être sûr que :

* s1 ≤ 50 (sinon il serait élu)
* s1 ≥ 12,5 (sinon il serait battu)

En testant `s1 < 12,5` **avant** le test de tête, on élimine les battus. Le `SinonSi` de la ligne 13 ne s'exécute que pour les candidats qualifiés.

---

|    11 | Sinon, je teste si s1 est inférieur à 12,5. |

|    13 | Sinon, je teste si s1 est le plus grand (ou égal) parmi les quatre scores. |

## Inspection des valeurs

### Exemple 1 — s1 = 55, s2 = 20, s3 = 15, s4 = 10

* `s1 > 50` : vrai → **élu**

### Exemple 2 — s1 = 8, s2 = 40, s3 = 30, s4 = 22

* `s1 > 50` : faux
* `s1 < 12,5` : vrai → **battu**

### Exemple 3 — s1 = 35, s2 = 30, s3 = 20, s4 = 15

* `s1 > 50` : faux
* `s1 < 12,5` : faux
* `s1 >= s2` : vrai, `s1 >= s3` : vrai, `s1 >= s4` : vrai → **ballottage favorable**

### Exemple 4 — s1 = 20, s2 = 35, s3 = 25, s4 = 20

* `s1 > 50` : faux
* `s1 < 12,5` : faux
* `s1 >= s2` : faux (20 < 35) → **ballottage défavorable**

### Exemple 5 (égalité) — s1 = 25, s2 = 25, s3 = 25, s4 = 25

* `s1 > 50` : faux
* `s1 < 12,5` : faux
* `s1 >= s2` : vrai, `s1 >= s3` : vrai, `s1 >= s4` : vrai → **ballottage favorable** ✓

L'égalité est bien traitée comme favorable, conformément à l'énoncé.

### Exemple 6 (borne exacte) — s1 = 50, s2 = 25, s3 = 15, s4 = 10

* `s1 > 50` : faux (50 n'est pas **strictement** supérieur à 50)
* `s1 < 12,5` : faux
* `s1 >= s2` : vrai, etc. → **ballottage favorable**

Un candidat à exactement 50 % n'est **pas** élu (il faut strictement plus de 50). Il est en ballottage.

---

## Conclusion

L'algorithme distingue les quatre issues en trois tests enchaînés :

1. élu (> 50 %)
2. battu (< 12,5 %)
3. ballottage favorable (qualifié et en tête, égalité incluse)
4. ballottage défavorable (qualifié mais pas en tête)

L'ordre des tests est important : on élimine d'abord les cas extrêmes (élu, battu) avant d'analyser la position relative du candidat.
