# Exercice 4.5 — Version 3 (battu / élu / ballottage favorable / défavorable)

## Introduction

On analyse **uniquement** le candidat 1, à partir des scores (en %) de 4 candidats au premier tour.

Résultats possibles :

* **élu**
* **battu**
* **ballottage favorable**
* **ballottage défavorable**

Règles :

* **Strictement plus de 50** → élu au premier tour.
* Sinon, second tour. Pour participer : **au moins 12,5**.
* **Ballottage favorable** = qualifié au second tour **et** arrivé en tête (**égalité incluse**).

Cette version distingue enfin les **4 issues**.

---

## Ce qu’il faut clarifier : “arrivé en tête” avec 4 candidats

L’énoncé précise : **“y compris en cas d’égalité”**.

Donc “en tête” signifie :

* le score de 1 est **supérieur ou égal** à celui de chaque autre candidat

Ce qui s’écrit :

`s1 >= s2 ET s1 >= s3 ET s1 >= s4`

Si un autre candidat a le même score, le candidat 1 est quand même considéré “en tête” (c’est le choix de la règle).

## Remarque sur l’ordre des tests

On teste d’abord les cas “extrêmes” :

1. `s1 > 50` → élu
2. `s1 < 12,5` → battu

Ensuite seulement, on teste “en tête”.

Pourquoi cet ordre ?
Parce que le test “en tête” (comparaison avec les autres) n’a de sens **que si** le candidat 1 est bien dans la zone :

* pas élu (donc `s1 <= 50`)
* pas battu (donc `s1 >= 12,5`)

En éliminant “élu” puis “battu” avant, on garantit que le test “en tête” ne s’applique qu’aux candidats **en ballottage**.

---

## Lecture en français

| Test                               | Ce que ça veut dire                       | Résultat               |
| ---------------------------------- | ----------------------------------------- | ---------------------- |
| `s1 > 50`                          | strictement plus de 50                    | élu                    |
| `s1 < 12,5`                        | pas qualifié                              | battu                  |
| `s1 >= s2 ET s1 >= s3 ET s1 >= s4` | qualifié **et** en tête (égalité incluse) | ballottage favorable   |
| sinon                              | qualifié mais pas en tête                 | ballottage défavorable |

---

## Inspection des valeurs

### Exemple 1 — s1 = 55, s2 = 20, s3 = 15, s4 = 10

* `s1 > 50` : vrai → **élu**

### Exemple 2 — s1 = 8, s2 = 40, s3 = 30, s4 = 22

* `s1 > 50` : faux
* `s1 < 12,5` : vrai → **battu**

### Exemple 3 — s1 = 35, s2 = 30, s3 = 20, s4 = 15

* pas élu, pas battu
* `s1 >= s2`, `s1 >= s3`, `s1 >= s4` : vrai → **ballottage favorable**

### Exemple 4 — s1 = 20, s2 = 35, s3 = 25, s4 = 20

* pas élu, pas battu
* `s1 >= s2` : faux → **ballottage défavorable**

### Exemple 5 (égalité) — s1 = 25, s2 = 25, s3 = 25, s4 = 25

* pas élu, pas battu
* toutes les comparaisons `>=` : vrai → **ballottage favorable** ✓

### Exemple 6 (borne exacte) — s1 = 50, s2 = 25, s3 = 15, s4 = 10

* `s1 > 50` : faux
* pas battu
* en tête : vrai → **ballottage favorable**

Un candidat à **50** n’est pas élu : il faut **strictement plus de 50**.

## Conclusion

Cette version distingue proprement les 4 issues :

1. **élu** (> 50)
2. **battu** (< 12,5)
3. **ballottage favorable** (qualifié + en tête, égalité incluse)
4. **ballottage défavorable** (qualifié + pas en tête)

Et l’affichage est centralisé : **un seul `Ecrire` final**.
