# Exercice 3.1 — Version 1 (sans `Sinon`)

## Introduction

Ici, on écrit une première version simple : on fait un seul test.

L’idée est volontairement basique pour qu’on puisse ensuite voir le défaut et l’améliorer dans une version suivante.

On ne traite pas le zéro (on suppose que l’utilisateur n’entre pas 0).

---

## Objectifs

Comprendre que :

* un `Si` exécute un bloc uniquement si la condition est vraie ;
* si on ne met pas de `Sinon`, il y a un cas où rien n’est affiché.

---

## Pseudo-code

```
1   Variables n en Numérique
2
3   Début
4       Ecrire "Entrez un nombre (non nul) : "
5       Lire n
6
7       Si n > 0 Alors
8           Ecrire "Le nombre est positif."
9       FinSi
10  Fin
```

---

## Syntaxe

* Condition : `n > 0`

  * vraie si `n` est positif ;
  * fausse si `n` est négatif (et aussi si `n` vaut 0, mais ici on l’ignore).

* Test sans `Sinon` :

  * si la condition est vraie, on affiche quelque chose ;
  * si la condition est fausse, on n’affiche rien.

---

## Lecture en français

| Ligne | Action décrite                                                |
| ----: | ------------------------------------------------------------- |
|     1 | Je déclare une variable `n`.                                  |
|     4 | Je demande un nombre non nul.                                 |
|     5 | Je lis ce nombre et je le stocke dans `n`.                    |
|     7 | Je teste la condition `n > 0`.                                |
|     8 | Si la condition est vraie, j’affiche “Le nombre est positif.” |
|     9 | Je termine le test.                                           |
|    10 | Je termine l’algorithme.                                      |

---

Problème : -3 est négatif, donc `n > 0` est fausse, et comme il n’y a pas de `Sinon`, l’algorithme n’affiche rien.

---

## Défaut de cette version

Cette version ne répond pas à la consigne complète :

* elle indique “positif” si le nombre est positif ;
* mais elle n’indique rien si le nombre est négatif.

L’utilisateur peut croire que l’algorithme ne fonctionne pas, alors qu’il manque simplement un traitement pour l’autre cas.

---

## Conclusion

La version 1 est une première étape :

* elle montre comment tester `n > 0` ;
* mais elle est incomplète car elle ne traite pas le cas négatif.

Une [version suivante](3.1_v2.md) devra aussi afficher un message quand la condition est fausse.
