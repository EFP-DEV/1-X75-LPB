# Exercice 3.3 — Version 1 (naïve, comparaison partielle)

## Introduction

On doit demander trois noms à l’utilisateur et indiquer s’ils sont triés dans l’ordre alphabétique.

Les noms identiques sont considérés comme triés, donc l’ordre autorise l’égalité.

Pour commencer, on propose une première version simple, puis on verra son défaut.

---

## Objectifs

Comprendre que :

* trier “dans l’ordre alphabétique” signifie : le 1er nom doit être avant (ou égal) au 2e, et le 2e avant (ou égal) au 3e ;
* comparer des textes se fait avec des opérateurs de comparaison (`<`, `<=`, etc.) en pseudo-code (comparaison lexicographique) ;
* une version naïve peut oublier une condition et donner de mauvais résultats.

---

## Pseudo-code (Version 1)

```
1   Variables n1, n2, n3 en Chaîne
2
3   Début
4       Ecrire "Entrez le premier nom : "
5       Lire n1
6       Ecrire "Entrez le deuxième nom : "
7       Lire n2
8       Ecrire "Entrez le troisième nom : "
9       Lire n3
10
11      Si n1 <= n2 Alors
12          Ecrire "Les noms sont triés."
13      Sinon
14          Ecrire "Les noms ne sont pas triés."
15      FinSi
16  Fin
```

---

## Lecture en français

| Ligne | Action décrite                                |
| ----: | --------------------------------------------- |
|     4 | Je demande le premier nom.                    |
|     5 | Je lis `n1`.                                  |
|     6 | Je demande le deuxième nom.                   |
|     7 | Je lis `n2`.                                  |
|     8 | Je demande le troisième nom.                  |
|     9 | Je lis `n3`.                                  |
|    11 | Je teste si `n1` est avant ou égal à `n2`.    |
|    12 | Si la condition est vraie, j’affiche “triés”. |
|    14 | Sinon, j’affiche “non triés”.                 |

---

## Inspection des valeurs

### Exemple 1 — n1 = "Alice", n2 = "Bernard", n3 = "Charles"

* `n1 <= n2` est vraie
  Affichage : Les noms sont triés.

Ici, c’est correct.

### Exemple 2 — n1 = "Alice", n2 = "Charles", n3 = "Bernard"

* `n1 <= n2` est vraie
  Affichage : Les noms sont triés.

Ici, c’est faux : “Alice, Charles, Bernard” n’est pas trié car `n2` et `n3` sont dans le mauvais ordre.

---

## Défaut de cette version

Cette version ne vérifie qu’une partie de la condition :

* elle vérifie `n1 <= n2`
* mais elle ne vérifie pas `n2 <= n3`

Donc elle peut annoncer “trié” alors que le troisième nom casse l’ordre.

---

## Conclusion

La version 1 est incomplète : comparer seulement les deux premiers noms ne suffit pas.

Une version suivante devra vérifier les deux comparaisons nécessaires :

* `n1 <= n2`
* et `n2 <= n3`

pour pouvoir conclure que les trois noms sont triés.
