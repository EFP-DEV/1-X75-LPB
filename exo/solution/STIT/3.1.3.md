# Exercice 3.1 — Version 3

## Introduction

Dans cette version, on utilise `Sinon` pour traiter clairement les deux possibilités :

* le nombre est positif ;
* le nombre n’est pas positif.

Mais attention : “n’est pas positif” signifie deux choses possibles :

* négatif ;
* égal à 0.

Or, la consigne disait au départ qu’on ne traite pas 0. Ici, on va expliquer ce qui se passe si 0 apparaît quand même.

---

## Objectifs

Comprendre que :

* `Sinon` correspond au cas où la condition est fausse ;
* si la condition est `n > 0`, alors `Sinon` couvre à la fois `n < 0` et `n = 0` ;
* si on veut distinguer “négatif” de “zéro”, il faut un test supplémentaire.

---

## Pseudo-code (Version 3)

```
1   Variables n en Numérique
2
3   Début
4       Ecrire "Entrez un nombre : "
5       Lire n
6
7       Si n > 0 Alors
8           Ecrire "Le nombre est positif."
9       Sinon
10          Ecrire "Le nombre est négatif."
11      FinSi
12  Fin
```

| Ligne | Action décrite                                             |

---

## Inspection des valeurs

### Exemple 1 — si l’utilisateur tape 7

* `n > 0` est vraie
  Affichage : Le nombre est positif.

### Exemple 2 — si l’utilisateur tape -3

* `n > 0` est fausse
  Affichage : Le nombre est négatif.

### Exemple 3 — si l’utilisateur tape 0

* `n > 0` est fausse
  Affichage : Le nombre est négatif.

---

## Explication du cas du 0

Le test est :

* `n > 0`

Donc :

* si `n` vaut 7, la condition est vraie, on affiche “positif” ;
* si `n` vaut -3, la condition est fausse, on passe dans `Sinon`, on affiche “négatif” ;
* si `n` vaut 0, la condition est aussi fausse, on passe dans `Sinon`, on affiche “négatif”.

Cela signifie que cette version “range” le 0 dans le même cas que les nombres négatifs, simplement parce que le `Sinon` signifie : “tous les autres cas”.

---

## Défaut de cette version

Cette version n’est correcte que si on garantit que l’utilisateur n’écrit jamais 0.

Si 0 est possible, l’affichage devient faux, car 0 n’est ni positif ni négatif.

Une version suivante devra distinguer 3 cas :

* `n > 0` : positif
* `n < 0` : négatif
* sinon : zéro

---

## Conclusion

Cette version est plus simple que la version 2 (un seul test avec `Sinon`), mais elle a un défaut important : si l’utilisateur saisit 0, l’algorithme affiche “négatif”.

Pour corriger cela, il faudra ajouter un traitement spécifique du zéro.
