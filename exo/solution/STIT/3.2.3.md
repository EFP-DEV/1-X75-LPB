# Exercice 3.2 — Version 2 (sans calculer le produit, avec `Sinon`)

## Introduction

Dans cette version, on respecte la contrainte : on ne calcule pas `a * b`.

On utilise une règle simple :

* un produit est positif si les deux nombres ont le même signe ;
* un produit est négatif si les deux nombres ont des signes différents.

On ignore le cas où le produit est nul, donc on suppose que `a` et `b` ne valent pas 0.

---

## Objectifs

Comprendre que :

* `a > 0` signifie “a est positif”, `a < 0` signifie “a est négatif” ;
* deux nombres de même signe donnent un produit positif ;
* deux nombres de signes opposés donnent un produit négatif ;
* on peut coder cela sans calculer le produit.

---

## Pseudo-code

```
1   Variables a, b en Numérique
2
3   Début
4       Ecrire "Entrez un premier nombre (non nul) : "
5       Lire a
6       Ecrire "Entrez un second nombre (non nul) : "
7       Lire b
8
9       Si (a > 0 ET b > 0) OU (a < 0 ET b < 0) Alors
10          Ecrire "Le produit est positif."
11      Sinon
12          Ecrire "Le produit est négatif."
13      FinSi
14  Fin
```

---

## Syntaxe

* `ET` : les deux conditions doivent être vraies.
* `OU` : au moins une des deux conditions doit être vraie.
* `( ... )` : sert à regrouper les conditions.

Condition de la ligne 9 :

* `(a > 0 ET b > 0)` : les deux sont positifs
* `(a < 0 ET b < 0)` : les deux sont négatifs
  Dans ces deux cas, le produit est positif.

Le `Sinon` couvre tous les autres cas, donc ici les signes opposés (et aussi le cas du zéro si jamais il arrive).

---

## Lecture en français

| Ligne | Action décrite                                  |
| ----: | ----------------------------------------------- |
|     4 | Je demande un premier nombre non nul.           |
|     5 | Je lis `a`.                                     |
|     6 | Je demande un second nombre non nul.            |
|     7 | Je lis `b`.                                     |
|     9 | Je teste si `a` et `b` ont le même signe.       |
|    10 | Si la condition est vraie, j’affiche “positif”. |
|    12 | Sinon, j’affiche “négatif”.                     |

---

## Inspection des valeurs

### Exemple 1 — a = 2, b = 3

* `(a > 0 ET b > 0)` est vraie
* la condition globale est vraie
  Affichage : Le produit est positif.

### Exemple 2 — a = -2, b = -5

* `(a < 0 ET b < 0)` est vraie
* la condition globale est vraie
  Affichage : Le produit est positif.

### Exemple 3 — a = -2, b = 3

* les deux sous-conditions sont fausses
* la condition globale est fausse
  Affichage : Le produit est négatif.

---

## Défaut restant

Cette version suppose que `a` et `b` sont non nuls.

Si l’utilisateur saisit 0 :

* la condition de “même signe” devient fausse
* on tombe dans `Sinon` et on affiche “négatif”
* ce serait faux, car le produit serait nul

Une version suivante pourra traiter explicitement le cas `a = 0` ou `b = 0` avant de décider.

---

## Conclusion

Cette version respecte la contrainte : on ne calcule pas le produit.

Elle détermine le signe du produit uniquement avec des comparaisons :

* même signe → produit positif
* signes opposés → produit négatif

Elle reste incorrecte si 0 est saisi, ce qui sera corrigé dans une version suivante.
